IoT Temperature Monitoring System

Immutable Monitoring via QBFT Blockchain (ERM-1 & ERM-2 Units)

System Overview

This project implements a decentralized monitoring system using QBFT (Quorum Byzantine Fault Tolerant) blockchain technology. Two independent hardware units communicate via a blockchain network to ensure tamper-proof environmental records.

Decentralized

QBFT consensus ensures data integrity across 5 validator nodes.

Immutable

Data stored permanently on the blockchain, protected from modification.

Key Components

  • ERM-1: Hardware Sender Unit (Temperature Simulation & Upload)
  • ERM-2: Hardware Receiver Unit (Data Retrieval & Alert Display)
  • GoQuorum Network: 5-node private blockchain
  • Backend Bridge: Node.js API server

Installation Guide

1. Node.js Setup

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

2. GoQuorum (Geth) Installation

# Extract and move binaries
sudo mv geth /usr/local/bin/
sudo mv bootnode /usr/local/bin/
geth version

Running the System

Step 1: Start Blockchain Node 0

cd ~/QBFT-Network/Node-0
geth --datadir data --networkid 1337 --istanbul.blockperiod 5 \
--http --http.port 22000 --http.api admin,eth,debug,miner,istanbul \
--unlock ${ADDRESS} --allow-insecure-unlock

Step 2: Start Backend Bridge

cd ~/server/iot-blockchain-backend
node server.js
Status: Server listening on port 3000.
